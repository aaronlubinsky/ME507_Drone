void armESC ( void )

Arms the ESCs with a minimum throttle signal.

This function initializes the ESCs so they are ready to receive throttle commands. It must be called before any attempt to update motor speeds.

Arms the ESCs with a minimum throttle signal.

This function sends the proper PWM signals to arm all four ESC motors. It starts PWM generation on all channels and sends a specific pulse width (approximately 1000Î¼s) to arm the ESCs. The function continues until the roll_set value reaches a threshold, indicating the system is ready.

Note
This function blocks execution until arming is complete
Warning
Ensure motors are properly secured before calling this function

◆ update_Motors()

void update_Motors ( void )

Updates the motor speeds based on control inputs.

This function should be called in the main control loop to continuously update motor PWM signals.

Updates the motor speeds based on control inputs.

This function performs the main control loop operation:

  1. Calculates PID errors for roll, pitch, and yaw axes
  2. Computes PID control efforts for each axis
  3. Mixes control efforts to determine individual motor speeds
  4. Applies safety limits and updates PWM outputs

The motor mixing follows a standard quadcopter X-configuration:

  • Motor A (front-right): +pitch, -roll, -yaw
  • Motor B (rear-right): -pitch, -roll, +yaw
  • Motor C (rear-left): -pitch, +roll, -yaw
  • Motor D (front-left): +pitch, +roll, +yaw
Note
PWM range: 960 (0% throttle) to 2000 (100% throttle)

Function should be called at regular intervals (typically 1kHz)
